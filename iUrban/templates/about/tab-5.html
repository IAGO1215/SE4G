Requirements and domain assumptions

Functional Requirements

Functional Requirements

You can view comments, leave a comment and delete a comment

Non-Functional Requirements

The website must not put at risk the user while browsing or downloading the information; The system shall always provide a safe environment which ensures that the downloaded content is virus and malware free; The web application shall be a safe environment regarding data veracity; Each part of the project will be tested as well as each of the components; The project is divided into several layers and components, due to this the maintainability will be high; The system shall be scalable, with the ability of extending to countries or areas that currently do not have service.

Technical Requirements

The program language must be Python; The web page language must be HTML, CSS, JS; The function interface should be Easy; The web page display language should be English; The system should use PostgreSQL Database to store the information.
